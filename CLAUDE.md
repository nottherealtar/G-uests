## Project Overview

**Guests - Game Quest Player** (by Tar) is a Python GUI application that lets users complete Discord Quests without downloading full games. It spoofs game presence to Discord by copying a lightweight placeholder executable to the path Discord expects for a given game, then launching it.

## Running the Application

**Install dependencies:**
```bash
pip install requests beautifulsoup4 Pillow
```

**Run:**
```bash
python questHunter.py
```

On first run with `main.mfs` missing, the app auto-compiles `gif.py` using PyInstaller (requires `pip install pyinstaller`).

## Publishing / Repo Setup

This project is meant to live at `github.com/YOUR_USERNAME/G-uests`. After creating that repo:

```bash
git remote set-url origin https://github.com/YOUR_USERNAME/G-uests.git
git push -u origin main
```

Then update `GITHUB_USER` in `questHunter.py` to match your username.

## Architecture

### Core Flow (`questHunter.py`)

1. **Remote-first fetch**: Tries `RAW_BASE/index.html` (raw GitHub URL). On success, caches to `Data-Feed/index.html`. Falls back to local cache if remote fails.
2. **HTML parsing**: BeautifulSoup parses quest entries. Each `<div data-name="...">` has a banner `<img>` and an `<a href="#<Windows\path\to\exe">`.
3. **Game selector UI**: Compact list of game names on top; clicking shows the banner and a Launch button below.
4. **Launch handler**: Validates the href path stays inside `Binaries/Win64/` (path traversal check), copies `main.mfs` → target `.exe`, copies `source.mui`, launches via `os.startfile()`.

### Key Constants (`questHunter.py`)
- `GITHUB_USER` / `REPO_NAME` — set after pushing to your own repo
- `RAW_BASE` — derived raw.githubusercontent.com URL for the data feed
- `ROOT` — `Binaries/Win64/` relative to cwd, where game stubs are placed

### Required Template Files (not committed)
- `Binaries/Win64/main.mfs` — compiled `gif.py` exe (auto-built on first run if `gif.py` present)
- `Binaries/Win64/en-US/source.mui` — empty placeholder

### Automated Quest Data Feed

The data feed is fully automated via GitHub Actions:

**`.github/workflows/update-quests.yml`** — runs every 4 hours (and on manual dispatch):
1. Calls `Discord API /quests` (user auth via `DISCORD_TOKEN` secret)
2. Calls `/applications/detectable` (public, no auth)
3. Generates banner PNGs from Discord CDN icons
4. Writes `Data-Feed/index.html` (with `data-name` attributes) and `Data-Feed/quests.json`
5. Commits and pushes if anything changed

**`.github/scripts/update_feed.py`** — the Python script that does all the above.

### Setting Up the GitHub Action

1. Go to your repo → **Settings → Secrets → Actions**
2. Add secret `DISCORD_TOKEN` = your personal Discord user token (not a bot token)
   - Get it from the Discord desktop app: open DevTools → Application → Local Storage → `https://discord.com` → `token`
3. The action runs automatically every 4 hours, or trigger it manually from the Actions tab

### Data Feed (`Data-Feed/`)
- `index.html` — quest list; auto-generated by the GitHub Action. Can be manually edited for overrides.
- `quests.json` — structured metadata (app_id, exe name, href) for each active quest
- `Assets/` — banner PNGs named by Discord `app_id` (e.g. `356875221078245376.png`) plus `button.png`

### Adding a Quest Manually
Add a `<div>` to `Data-Feed/index.html`:
```html
<div data-name="Game Name" style="padding-bottom:20px;padding-left:12px;">
    <img src="./Assets/YOUR_BANNER.png" alt="Game Name" style="display:block;">
    <a href="#Steam\steamapps\common\GameName\Binaries\Win64\ExeStem">
        <img src="./Assets/button.png" alt="">
    </a>
</div>
```
The `href` after `#` is the Windows path that will be created under `Binaries/Win64/`.

### `gif.py`
Standalone tkinter app (shows an animated GIF). Compiled into `main.mfs` — the stub that becomes each spoofed game executable. Very large file due to embedded base64 assets.
